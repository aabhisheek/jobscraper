Explain code at 5 layers of depth.

Target: $ARGUMENTS (file path, function name, or module name).

Layer 1 — One sentence: what does this code do?
Layer 2 — How it fits: where is it called from, what does it call?
Layer 3 — Step by step: walk through the logic line by line.
Layer 4 — Design: why was it built this way? What alternatives exist?
Layer 5 — Edge cases: what can go wrong? What are the assumptions?
